## 1. 数据库概述

- **数据库类型：** MySQL 8.0（或 PostgreSQL 14+）
- **字符集：** utf8mb4（支持 emoji）
- **时区：** Asia/Shanghai（UTC+8）
- **存储引擎：** InnoDB
- **大小写：** lowercase_table_names = 1

---

## 2. 命名规范

| 实体 | 命名规则 | 示例 |
|------|---------|------|
| 表名 | `sft_{模块}_{实体}`，全小写，下划线分隔 | `sft_customer` |
| 字段名 | `snake_case`，全小写 | `customer_name` |
| 主键 | `{表名}_id` | `customer_id` |
| 外键 | `{引用表名}_id` | `customer_id` |
| 创建时间 | `created_at` | `created_at` |
| 更新时间 | `updated_at` | `updated_at` |
| 逻辑删除 | `deleted_at`（NULL = 未删除） | `deleted_at` |
| 枚举值 | 统一用字符串存储 | `status VARCHAR(20)` |

### 3.1 用户与权限（`sft_auth_*`）

#### 3.1.1 用户表 `sft_auth_user`

```sql
CREATE TABLE sft_auth_user (
    user_id          VARCHAR(36)  PRIMARY KEY,          -- UUID
    username         VARCHAR(50)  NOT NULL UNIQUE,
    password_hash    VARCHAR(255) NOT NULL,
    real_name        VARCHAR(100) NOT NULL,
    role             ENUM('admin','director','finance','sales') NOT NULL,
    email            VARCHAR(255),
    phone            VARCHAR(20),
    status           ENUM('active','inactive') DEFAULT 'active',
    created_at       DATETIME     DEFAULT CURRENT_TIMESTAMP,
    updated_at       DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted_at       DATETIME     DEFAULT NULL,
    INDEX idx_username (username),
    INDEX idx_role (role),
    INDEX idx_deleted_at (deleted_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

| 字段 | 类型 | 说明 |
|------|------|------|
| user_id | VARCHAR(36) | UUID 主键 |
| username | VARCHAR(50) | 登录用户名 |
| password_hash | VARCHAR(255) | 密码（bcrypt 加密） |
| real_name | VARCHAR(100) | 真实姓名 |
| role | ENUM | 角色：admin/director/finance/sales |
| email | VARCHAR(255) | 邮箱 |
| phone | VARCHAR(20) | 电话 |
| status | ENUM | 账号状态 |

---

#### 3.1.2 角色权限表 `sft_auth_permission`

```sql
CREATE TABLE sft_auth_permission (
    permission_id   VARCHAR(36) PRIMARY KEY,
    code            VARCHAR(100) NOT NULL UNIQUE,  -- 如 'forecast.submit'
    name            VARCHAR(100) NOT NULL,
    module          VARCHAR(50)  NOT NULL,         -- 如 'forecast'
    description     VARCHAR(255),
    created_at      DATETIME     DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### 3.1.3 角色-权限关联表 `sft_auth_role_permission`

```sql
CREATE TABLE sft_auth_role_permission (
    role            VARCHAR(20)  NOT NULL,  -- admin/director/finance/sales
    permission_id   VARCHAR(36)  NOT NULL,
    PRIMARY KEY (role, permission_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3.2 基础数据（`sft_base_*`）

#### 3.2.1 客户表 `sft_base_customer`

```sql
CREATE TABLE sft_base_customer (
    customer_id      VARCHAR(36) PRIMARY KEY,
    customer_code    VARCHAR(50) NOT NULL UNIQUE,
    customer_name    VARCHAR(200) NOT NULL,
    customer_type    ENUM('direct','channel','end_user') DEFAULT 'direct',
    contact_name     VARCHAR(100),
    contact_phone    VARCHAR(20),
    contact_email    VARCHAR(255),
    status           ENUM('active','inactive') DEFAULT 'active',
    created_at       DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at       DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted_at       DATETIME DEFAULT NULL,
    INDEX idx_code (customer_code),
    INDEX idx_name (customer_name),
    INDEX idx_deleted_at (deleted_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### 3.2.2 开票公司表 `sft_base_invoice_company`

```sql
CREATE TABLE sft_base_invoice_company (
    invoice_company_id   VARCHAR(36) PRIMARY KEY,
    company_code        VARCHAR(50) NOT NULL UNIQUE,
    company_name        VARCHAR(200) NOT NULL,
    tax_number          VARCHAR(50),
    address             VARCHAR(500),
    performance_attr    VARCHAR(100),          -- 业绩归属
    sales_region        VARCHAR(100),          -- 销售大区
    status              ENUM('active','inactive') DEFAULT 'active',
    created_at          DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at          DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted_at          DATETIME DEFAULT NULL,
    INDEX idx_performance (performance_attr),
    INDEX idx_region (sales_region),
    INDEX idx_deleted_at (deleted_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### 3.2.3 产品分类表 `sft_base_product_category`

```sql
CREATE TABLE sft_base_product_category (
    category_id     VARCHAR(36) PRIMARY KEY,
    category_code   VARCHAR(50) NOT NULL,
    category_name   VARCHAR(200) NOT NULL,
    parent_id       VARCHAR(36) DEFAULT NULL,    -- NULL = 一级
    level           TINYINT NOT NULL DEFAULT 1, -- 1/2/3/4
    path            VARCHAR(500),                -- 如 'Holemaking/整体硬质合金钻头/3xD'
    sort_order      INT DEFAULT 0,
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted_at      DATETIME DEFAULT NULL,
    INDEX idx_parent (parent_id),
    INDEX idx_path (path(255)),
    INDEX idx_deleted_at (deleted_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

> **说明：** 产品分类使用邻接表模型，通过 `parent_id` 和 `path` 两种方式支持树形查询。

#### 3.2.4 产品表 `sft_base_product`

```sql
CREATE TABLE sft_base_product (
    product_id      VARCHAR(36) PRIMARY KEY,
    product_code    VARCHAR(50) NOT NULL UNIQUE,
    product_name    VARCHAR(200) NOT NULL,
    pa              VARCHAR(100) NOT NULL,       -- PA 产品线（来自分类 path 一级）
    sub_pa_1        VARCHAR(100) DEFAULT '',
    sub_pa_2        VARCHAR(100) DEFAULT '',
    sub_pa_3        VARCHAR(100) DEFAULT '',
    sub_pa_4        VARCHAR(100) DEFAULT '',
    spec            VARCHAR(200),                -- 规格
    unit            VARCHAR(20),                -- 单位：件/把/套/片
    std_price       DECIMAL(15,2) DEFAULT 0,     -- 标准单价（元）
    category_id     VARCHAR(36),                 -- 关联产品分类
    status          ENUM('active','inactive') DEFAULT 'active',
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted_at      DATETIME DEFAULT NULL,
    INDEX idx_pa (pa),
    INDEX idx_category (category_id),
    INDEX idx_deleted_at (deleted_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### 3.2.5 产品价格规则表 `sft_base_product_pricing`

```sql
CREATE TABLE sft_base_product_pricing (
    pricing_id          VARCHAR(36) PRIMARY KEY,
    product_id          VARCHAR(36) NOT NULL,
    invoice_company_id  VARCHAR(36),             -- NULL = 通用价格
    customer_id         VARCHAR(36),              -- NULL = 通用价格（所有客户）

    start_date          DATE NOT NULL,
    end_date            DATE,
    status              ENUM('active','future','expired') DEFAULT 'active',
    created_at          DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at          DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted_at          DATETIME DEFAULT NULL,
    INDEX idx_product (product_id),
    INDEX idx_invoice (invoice_company_id),
    INDEX idx_customer (customer_id),
    INDEX idx_dates (start_date, end_date),
    INDEX idx_deleted_at (deleted_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### 3.2.6 区域/业绩归属映射表 `sft_base_region_mapping`

```sql
CREATE TABLE sft_base_region_mapping (
    mapping_id          VARCHAR(36) PRIMARY KEY,

    created_at          DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at          DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted_at          DATETIME DEFAULT NULL,
    INDEX idx_region (sales_region),
    INDEX idx_director (director_user_id),
    INDEX idx_deleted_at (deleted_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3.3 预测业务（`sft_forecast_*`）

#### 3.3.1 预测周期表 `sft_forecast_period`

```sql
CREATE TABLE sft_forecast_period (
    period_id       VARCHAR(36) PRIMARY KEY,
    period_name     VARCHAR(50) NOT NULL UNIQUE,  -- 如 2026FC1
    open_start_date DATE NOT NULL,
    open_end_date   DATE NOT NULL,
    months          JSON NOT NULL,               -- ["Jan","Feb","Mar","Apr"]
    status          ENUM('open','history','closed') DEFAULT 'open',
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted_at      DATETIME DEFAULT NULL,
    INDEX idx_status (status),
    INDEX idx_deleted_at (deleted_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### 3.3.2 预测明细表 `sft_forecast_record`

```sql
CREATE TABLE sft_forecast_record (
    record_id           VARCHAR(36) PRIMARY KEY,
    period_id           VARCHAR(36) NOT NULL,
    user_id             VARCHAR(36) NOT NULL,       -- 填报人
    customer_id         VARCHAR(36) NOT NULL,
    invoice_company_id  VARCHAR(36) NOT NULL,
    performance_attr    VARCHAR(100) NOT NULL,
    sales_region        VARCHAR(100) NOT NULL,
    pa                  VARCHAR(100) NOT NULL,
    sub_pa_1           VARCHAR(100) DEFAULT '',
    sub_pa_2           VARCHAR(100) DEFAULT '',
    sub_pa_3           VARCHAR(100) DEFAULT '',
    sub_pa_4           VARCHAR(100) DEFAULT '',
    -- 每月预测数据（JSON 或拆分列）
    monthly_data        JSON NOT NULL,   -- {"Jan": {"qty": 100, "inv": 80}, "Feb": {...}, ...}
    approval_status     ENUM('draft','submitted_director','submitted_finance','approved','rejected')
                           DEFAULT 'draft',
    submitted_at        DATETIME DEFAULT NULL,
    approved_at         DATETIME DEFAULT NULL,
    created_at          DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at          DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted_at          DATETIME DEFAULT NULL,
    INDEX idx_period (period_id),
    INDEX idx_user (user_id),
    INDEX idx_customer (customer_id),
    INDEX idx_invoice (invoice_company_id),
    INDEX idx_approval (approval_status),
    INDEX idx_deleted_at (deleted_at),
    INDEX idx_period_user (period_id, user_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

> **说明：** `monthly_data` JSON 结构示例：
> ```json
> {
>   "Jan": { "qty": 100, "inv": 80 },
>   "Feb": { "qty": 120, "inv": 95 },
>   "Mar": { "qty": 80,  "inv": 70 }
> }
> ```

#### 3.3.3 审批历史表 `sft_forecast_approval_history`

```sql
CREATE TABLE sft_forecast_approval_history (
    history_id      VARCHAR(36) PRIMARY KEY,
    record_id       VARCHAR(36) NOT NULL,
    action          ENUM('create','submit','director_approve','director_reject','finance_approve','finance_reject') NOT NULL,
    operator_id     VARCHAR(36) NOT NULL,
    opinion         VARCHAR(500) DEFAULT '',
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_record (record_id),
    INDEX idx_operator (operator_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 6. 分区策略

### 6.1 预测记录表 `sft_forecast_record`

按 `period_id` 或按 `created_at` 年度分区：

```sql
PARTITION BY LIST COLUMNS (period_name) (
    PARTITION p2025fc2 VALUES IN ('2025FC2'),
    PARTITION p2026fc1 VALUES IN ('2026FC1'),
    PARTITION p2026fc2 VALUES IN ('2026FC2'),
    PARTITION p_future VALUES IN (DEFAULT)
);
```

### 6.2 月度实际数据 `sft_monthly_actual`

按 `year_month` 年度 + 月分区：

```sql
PARTITION BY RANGE (year_month) (
    PARTITION p2025 VALUES LESS THAN ('2026-01'),
    PARTITION p2026 VALUES LESS THAN ('2027-01'),
    PARTITION p_future VALUES LESS THAN MAXVALUE
);
```

### 6.3 操作日志 `sft_log_operation`

按月分区，自动归档：

```sql
PARTITION BY RANGE (TO_DAYS(created_at)) (
    PARTITION p202604 VALUES LESS THAN (TO_DAYS('2026-05-01')),
    PARTITION p202605 VALUES LESS THAN (TO_DAYS('2026-06-01')),
    PARTITION p_future VALUES LESS THAN MAXVALUE
);
```
