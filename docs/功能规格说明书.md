## 1. 系统概述

### 1.1 项目背景

山高刀具（Sandvik）中国区需要一套数字化销售预测管理系统，帮助销售团队在线填报、审批和分析销售预测数据，提高预测准确率和协作效率。

### 1.2 系统目标

- 销售代表在线填报预测数据，减少 Excel 传递
- 多级审批流程（区域总监 → 财务）在线完成
- 实时数据看板，支持"订单"和"开票"两种口径切换
- 完整的基础数据管理（客户、产品、价格、区域映射）
- 历史数据追溯和趋势分析

### 1.3 技术栈

| 层次 | 技术 |
|------|------|
| 前端框架 | Vue 3 + antd-vue 4.x（CDN 单文件模式） |
| 图表 | ECharts 5 |
| 时间处理 | dayjs + 多语言插件 |
| 后端 | 预留接口（当前为 Mock 数据） |
| 部署 | 支持 Docker / 企业内网 |

## 2. 用户角色

| 角色 | 描述 | 权限范围 |
|------|------|---------|
| **销售人员（Sales）** | 填报和修改本人预测数据 | 填报、查看本人数据 |
| **区域总监（Director）** | 审批区域内的预测数据 | 审批、汇总查看 |

## 4. 业务流程

### 4.1 预测填报流程

```
[管理员] 创建预测周期（名称、时间、月份）
       ↓
[销售]  打开周期 → 下载模板 / 复制上期
       ↓
[销售]  填写或导入预测数据
       ↓
[销售]  保存草稿 / 提交审批
       ↓
[区总]  收到通知 → 审核 → 通过/拒绝
       ↓
[财务]  收到通知 → 最终审核 → 通过/拒绝
       ↓
[完成]  数据归档 → Dashboard 同步
```

### 4.2 审批流程配置

```
销售提交
   ↓
区总审批节点（指定审批人或角色）
   ↓
财务审批节点（指定审批人或角色）
   ↓
结束（通过）/ 返回修改
```

## 5. 接口设计

> **说明：** 当前版本前端使用 Mock 数据，以下为预留接口规范。

### 5.1 认证接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/auth/login` | 用户登录 |
| POST | `/api/auth/logout` | 登出 |

### 5.2 数据看板接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/dashboard/kpi?metric=order\|invoice&period=` | 获取 KPI |
| GET | `/api/dashboard/charts/{type}` | 获取图表数据 |

### 5.3 预测相关接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/forecast/periods` | 获取周期列表 |
| GET | `/api/forecast/rows?period=&filters=` | 获取预测明细 |
| POST | `/api/forecast/rows` | 新增预测记录 |
| PUT | `/api/forecast/rows/{id}` | 更新预测记录 |
| DELETE | `/api/forecast/rows/{id}` | 删除预测记录 |
| POST | `/api/forecast/submit` | 提交审批 |

### 5.4 审批相关接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/approval/list?status=&role=` | 获取审批列表 |
| POST | `/api/approval/approve/{id}` | 审批通过 |
| POST | `/api/approval/reject/{id}` | 审批拒绝 |
| GET | `/api/approval/history/{id}` | 获取审批历史 |

### 5.5 基础数据接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET/POST/PUT/DELETE | `/api/customers` | 客户 CRUD |
| GET/POST/PUT/DELETE | `/api/invoice-companies` | 开票公司 CRUD |
| GET/POST/PUT/DELETE | `/api/products` | 产品 CRUD |
| GET/POST/PUT/DELETE | `/api/pricing` | 价格规则 CRUD |
| GET/POST/PUT/DELETE | `/api/region-mappings` | 区域映射 CRUD |

## 6. 非功能需求

### 6.1 性能
- 页面首屏加载 ≤ 3秒
- 表格操作响应 ≤ 500ms

### 6.2 兼容性
- 支持 Chrome / Edge / Firefox 最新两个版本
- 响应式布局（支持 1080P + 移动端查看）

### 6.3 安全
- 登录态通过 Token 管理
- 敏感操作（删除/审批）需二次确认
- 操作日志完整记录

### 6.4 数据准确率
- 预测金额保留两位小数
- 所有金额单位统一为"万元"
