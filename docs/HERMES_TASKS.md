## 任务流转规则
1. Hermes 主动更新任务状态，不等 cron job
2. CC/小Q 任务委派后立即标 ⏳ 等待中
3. cron job 只负责检测 ⏳ 状态的任务是否完成，并更新状态
4. 阻塞任务由 Mark 解开后手动激活
5. 完成的任务保留 7 天后归档
6. 🚫 阻塞只阻塞自己，不阻塞其他任务（除非明确依赖）

## 调度规则（cron job 读取此文件）

```
每小时 cron job:
1. 扫描"⏳ 等待中"任务
   → 检查结果文件是否存在
   → 存在则读取结果、更新状态为✅或🚫
   → 结果异常则记录问题，重试或升级为🚫

2. 扫描"🔄 待处理"任务（按优先级）
   → 检查依赖是否满足
   → 依赖满足则激活任务
   → Hermes任务 → 立即执行
   → CC任务 → 委派CC
   → 小Q任务 → 委派小Q

3. 扫描"🚫 阻塞"任务
   → 检查阻塞原因是否解除
   → 解除则移回"🔄 待处理"

4. 不开新任务如果：
   → 有"⏳ 等待中"任务未决
   → 有"🔬 进行中"任务未完成
   → 有明确依赖当前阻塞任务的🔄待处理任务
   → 普通🚫阻塞任务 → 只阻塞自己，不挡路
```
