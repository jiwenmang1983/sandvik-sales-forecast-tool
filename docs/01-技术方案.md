# 技术方案

## 系统概述

Sandvik Forecast Tool 用于管理 Sandvik 产品销售预测流程，支持多用户协作、数据录入、审批管理和可视化分析。

## 技术架构

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   Vue 3     │────▶│   .NET 8    │────▶│   MySQL 8   │
│   Frontend  │     │   Web API   │     │   Database  │
└─────────────┘     └──────┬──────┘     └─────────────┘
                          │
                          ▼
                   ┌─────────────┐
                   │    Redis    │
                   │    Cache    │
                   └─────────────┘
```

## 前端技术选型

### Vue 3
- **理由**: 组合式 API (Composition API) 提供更好的代码组织，TypeScript 支持良好，生态系统成熟。

### Vite
- **理由**: 开发服务器启动快，HMR 热更新体验优秀，构建性能优于 webpack。

### Ant Design Vue 4
- **理由**: 企业级 UI 组件库，组件丰富、文档完善、与 Ant Design 设计语言一致。

### ECharts 5
- **理由**: 图表功能强大，支持多种图表类型，数据可视化能力突出。

## 后端技术选型

### .NET 8 / ASP.NET Core
- **理由**: 高性能跨平台框架，依赖注入完善，中间件生态丰富。

### Entity Framework Core 8
- **理由**: 成熟的 ORM，支持 LINQ 查询，迁移管理方便。

### Pomelo.EntityFrameworkCore.MySql
- **理由**: 最佳 MySQL EF Core provider，社区活跃，兼容性好。

## 数据库选型

### MySQL 8.0
- **理由**:
  - 支持 UTF8MB4 字符集，表情符号和特殊字符支持良好
  - 事务支持完整，数据一致性有保障
  - 运维成熟，备份恢复方案完善
  - 成本可控，适合企业级应用

## 部署方案

### 本地开发
- Docker Compose 编排 MySQL 和 Redis 服务
- .NET CLI dotnet run 启动 API
- npm run dev 启动前端开发服务器

### Azure Container App
- 前后端分离容器化部署
- Azure Database for MySQL 托管数据库
- Azure Cache for Redis 托管缓存
- 负载均衡和自动缩放

## 配套服务
