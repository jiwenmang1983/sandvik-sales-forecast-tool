## §3.2 预测周期管理（7个问题）

🔴 **1. 周期的"当前周期"如何确定** ✅已确认
系统里可能有多个历史周期（2025FC2、2026FC1），Dashboard 和填报页面显示哪个？有没有"当前激活周期"的概念，由谁设置（Admin手动？还是系统取最新的）？
→ **确认结论：按时间窗口自动锁定**。FillTimeStart/FillTimeEnd 时间窗口不重叠，同一时间只有唯一周期有效。系统按当前时间自动取落在窗口内的周期，无需IsActive字段。

🔴 **2. 填报窗口的时间校验边界** ✅已确认
填报截止判断用的是服务器时间还是浏览器本地时间？客户端和服务器时区差会不会导致本应截止的记录被错误提交？
→ **确认结论：统一用服务器时间（DateTime.UtcNow）**，不用浏览器时间。

🟢 **3. 预测月份范围的边界情况**
`period_start = 2026-01，period_end = 2026-01`，是1个月还是2个月？边界定义需要明确。

🔴 **4. 延期窗口和正常窗口的关系** ✅已确认
延期窗口是正常窗口的严格延长期（只有白名单可提交）？还是两个独立时间段？无延期信息时默认行为是什么？
→ **确认结论：两个独立窗口**。FillTimeEnd 锁定后，ExtensionStart 可灵活设定（=FillTimeEnd 或更晚），Extension窗口独立存在。ExtensionEnd = ExtensionStart + N天。

🔴 **5. 周期创建后能否修改字段** ✅已确认
周期已开始填报，Admin 能改哪些字段（名称？截止时间？）？改截止时间对已提交数据有什么影响？
→ **确认结论：Admin可改任意字段，但有数据的周期禁止删除**。底线：已有数据的周期不能删，字段可改。

🔴 **6. 周期结束后能否重新打开** ✅已确认
周期过了截止时间+延期时间，还能重新激活吗？数据怎么处理？
→ **确认结论：FillTimeEnd 过期后锁定，不可改。Extension 窗口可继续调整（通过调 ExtensionEnd 实现"延期重开"）。FillTimeEnd 本身不能改，只能用延期窗口来处理特殊情况。**

🟢 **7. 周期删除的限制** ✅已确认（隐含在Q5中）
如果周期已有预测数据，Admin 能删除周期吗？删除后数据怎么办？
→ **确认结论：底线——已有填报数据的周期禁止删除。字段可改，删除是禁区。**

## §3.3 填报流程（8个问题）

🔴 **8. 首次进入填报页面的默认状态** ✅已确认
首次进入时是空白表单？还是系统自动列出品牌匹配的客户供选择？有没有快速批量导入模式？
→ **确认结论：空白列表。显示已填报条目的数量和金额汇总。无数据时全空白。**

🔴 **9. 填报数据的自动保存** ✅已确认
如果销售填了很久浏览器崩溃，数据会不会丢？需不需要服务器端AutoSave？
→ **确认结论：不做 AutoSave。依赖模板上传/下载。**

🔴 **10. 草稿和正式提交的数据是否在同一个表** ✅已确认（补充确认）
Dashboard统计时包不包括草稿？审批流的数据口径怎么算？
→ **确认结论：同一表，Status区分。Dashboard和报表只含已审批通过数据，不含过程数据。**

🔴 **11. 填报时产品与客户的联动逻辑** ✅已确认
选了客户A后，产品下拉框是显示所有产品、品牌对应产品、还是该客户历史填报过的产品？
→ **确认结论：无联动，产品全部开放，选任意客户后产品下拉显示全部产品。**

🔴 **12. 填报月份列的展示逻辑** ✅已确认
月份列是横排（每月一列）还是竖排（一个月一行）？12个月是12列？
→ **确认结论：两种展示并存。单笔录入Form：月份是行（一行一个月份）。已填报数据查看：客户+产品为行，月份为列横排，Tab切换不同月。**

🔴 **13. 填报时的数据校验规则** ✅已确认
- 金额/数量：可为0（某些月份无数据），但**不能为负**（>= 0）
- 开票金额与订单金额无关联，各自分别独立填写，无上限/比例限制
- 月度数据与年度汇总无校验关系，各自独立

🟢 **14. 历史数据的只读控制**
被退回后是立刻可编辑还是等通知？直线经理审批中能看到编辑界面吗？

🔴 **15. 一个销售在同一周期同一客户同一产品能否产生多条记录**
这决定了数据模型的核心结构：每次提交是覆盖还是新增？

## §3.3 审批流程（9个问题）

🔴 **16. 审批推进机制：自动还是手动**
直线经理审批通过后，是系统自动推给上一级，还是需要经理手动再点一次？

🔴 **17. 审批链节点是否一定存在**
杨依柱（区域总监）"暂缺系统账号"，如果审批流需要区域总监但账号不存在，怎么处理？跳过？阻塞？

🔴 **18. 退回层级逻辑**
退回时是退到销售，还是退到直线经理（让经理重新审）？退回后原待审批记录怎么处理？

🔴 **19. 退回次数是否有限制**
被退回后修改再提交，再次被退回，如此循环有上限吗？

🔴 **20. Frank Tao的约束是前端还是后端**
"不能修改明细"是界面层按钮不显示，还是后端API强制校验？两种实现安全性差异很大。

🔴 **21. 调整动作后的数据重新提交流程**
销售修改数据后，是点"重新提交"重新走全链路，还是只回到被退回的那一级？

🟢 **22. 批量审批能力**
直线经理有10个下属批量提交时，是逐个处理还是支持"批量通过/驳回"？批量操作时调整值怎么应用？

🔴 **23. 审批撤回/撤销**
销售提交后经理还没审，销售能撤回吗？撤回后是作废还是变草稿？

🔴 **24. 审批历史的数据量和查询性能**
一个周期大量记录×多轮审批历史，ApprovalHistory表会不会过大？有索引吗？记录永久保留还是定期归档？

## §4.1 Dashboard（6个问题）

🔴 **25. Dashboard时间范围定义** ✅已确认（见Q14补充）
趋势图是当前周期内所有月份，还是最近N个月？跨品牌/区域时数据怎么聚合？
→ **确认结论：Dashboard永远显示最新有效数据——有活跃周期时显示当前周期，已全部关闭时显示最近一个有数据的周期。Dashboard不能空着。**

🔴 **26. Dashboard各角色数据范围一致性**
§4.1说"直线经理看本团队"，§5.2说"MANAGER看自己+管辖销售"——"本团队"和"管辖销售"等价吗？直线经理能看到自己的数据吗？

🔴 **27. Dashboard客户维度汇总的口径**
汇总的是预测金额、开票金额、还是两者？按客户维度展开还是按品牌？

🟢 **28. Dashboard数据统计口径和刷新频率**
只含已审批通过？含审批中？还是可选切换？实时查询还是定时汇总？审批状态变化后Dashboard多久更新？

🟢 **29. 空状态和加载状态**
无数据时显示什么？Loading骨架屏？API报错时友好提示？

🔴 **30. 未提交/草稿数据是否计入Dashboard** ✅已确认（与Q10关联）
销售提交了但经理还没审，直线经理的Dashboard能看到吗？
→ **确认结论：Dashboard和报表只含已审批通过数据。草稿、审批中的数据不计入。**
