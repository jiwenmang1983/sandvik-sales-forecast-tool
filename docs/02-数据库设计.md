# 数据库设计

## 命名规范

- **表名**: PascalCase，单数形式，如 `User`, `Customer`
- **字段名**: PascalCase，如 `UserName`, `CreatedAt`
- **主键**: `Id` (GUID)
- **外键**: `[表名]Id`，如 `UserId`, `CustomerId`
- **时间戳**: `CreatedAt`, `UpdatedAt`
- **逻辑删除**: `IsDeleted`

## 表结构设计

### Users (用户表)

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| Id | CHAR(36) | 用户唯一标识 | PRIMARY KEY |
| UserName | VARCHAR(50) | 用户名 | NOT NULL, UNIQUE |
| PasswordHash | VARCHAR(255) | 密码哈希 | NOT NULL |
| DisplayName | VARCHAR(100) | 显示名称 | NOT NULL |
| Email | VARCHAR(255) | 邮箱 | NOT NULL |
| Role | VARCHAR(20) | 角色 (Admin/Manager/User) | NOT NULL |
| IsActive | TINYINT(1) | 是否激活 | DEFAULT 1 |
| CreatedAt | DATETIME | 创建时间 | NOT NULL |
| UpdatedAt | DATETIME | 更新时间 | NOT NULL |
| IsDeleted | TINYINT(1) | 逻辑删除 | DEFAULT 0 |

### ProductHierarchy (产品层级表)

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| Id | CHAR(36) | 唯一标识 | PRIMARY KEY |
| ProductLevel | INT | 层级 (1=PA, 2=SubPA1, 3=SubPA2, 4=SKU) | NOT NULL |
| ParentId | CHAR(36) | 父级产品ID | FOREIGN KEY → ProductHierarchy.Id |
| ProductCode | VARCHAR(50) | 产品编码 | NOT NULL |
| ProductName | VARCHAR(200) | 产品名称 | NOT NULL |
| SortOrder | INT | 排序 | DEFAULT 0 |
| IsActive | TINYINT(1) | 是否激活 | DEFAULT 1 |
| CreatedAt | DATETIME | 创建时间 | NOT NULL |
| UpdatedAt | DATETIME | 更新时间 | NOT NULL |
| IsDeleted | TINYINT(1) | 逻辑删除 | DEFAULT 0 |

层级说明:
- Level 1: PA (Product Area)
- Level 2: Sub PA-1
- Level 3: Sub PA-2
- Level 4: SKU

## 索引设计

- `Users`: `UserName`, `Email`, `IsActive`
- `Customers`: `CustomerCode`, `SalesPersonId`
- `InvoiceCompanies`: `CompanyCode`
- `ProductHierarchy`: `ProductCode`, `ParentId`, `ProductLevel`
- `ForecastPeriods`: `PeriodYear`, `PeriodMonth`, `Status`
- `ForecastRecords`: `ForecastPeriodId`, `CustomerId`, `ProductId`, `(ForecastPeriodId, CustomerId, ProductId)` UNIQUE
- `Approvals`: `ForecastPeriodId`, `ApproverId`
- `LoginLogs`: `UserId`, `LoginTime`
- `OpLogs`: `UserId`, `CreatedAt`
